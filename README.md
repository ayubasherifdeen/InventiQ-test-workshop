  A test suite with of InventiQ frontend/saleperson side. It has about 9 test scenarios with
  over 90 test cases across board. 
  
  This test suite uses various techniques(negative and postive cases, boundary value analysis, etc)
  to derive functional, , ui and basic security test cases. It pays close attention and considers
  project requirements, as well as how users would normally expect the flow of this kind of app
  to be like.
  
  It also has a bug report of all bugs and defects identified with manual testing, recording about 17 bugs and defects.
  It details the actual and expected results of tests, records priority of bugs and defects and
  supports with screenshot evidence

  This test suit uses the traditional excell spreadshot and is meant to be my first project towards
  becoming a QA Engineer and fulfilling my passion for Quality

  PS: All test cases are executed with manual testing
